Page Replacement in Operating Systems

Understanding Memory Management Techniques

Why Page Replacement Matters

Page replacement is an important concept in operating systems that helps manage limited physical memory efficiently.

  • Allows programs larger than RAM to run using virtual memory.
  • Reduces unnecessary disk access by choosing the best page to replace.
  • Improves system responsiveness and multitasking capability.

Page Fault Handling Process

  1. A program requests access to a memory page.
  2. If the page is not present in RAM, a page fault occurs.
  3. The operating system searches for a free frame or selects a page to replace.
  4. If the selected page was modified, it is written back to disk.
  5. The requested page is loaded from secondary storage into memory.
  6. The program resumes execution as if nothing happened.

FIFO (First In First Out)

FIFO replaces the page that entered memory first.

  • Simple to implement using a queue data structure.
  • Pages are removed in the same order they were loaded.
  • Can suffer from Belady’s Anomaly, where more frames increase page faults.

Example

Reference String: 1, 2, 3, 4, 1

Frames: [1] → [1,2] → [1,2,3] → replace 1 → [4,2,3]

Page Faults: 4

Optimal Page Replacement

The Optimal algorithm replaces the page that will not be used for the longest time in the future.

  • Produces the minimum possible number of page faults.
  • Used mainly as a benchmark to compare other algorithms.
  • Not practical in real systems because future page references are unknown.

Example

Reference String: 7, 0, 1, 2, 0, 3

Frames start: [7,0,1] → next page = 2 → replace 7 → [2,0,1]

LRU (Least Recently Used)

LRU replaces the page that has not been used for the longest time in the past.

  • Based on the assumption that recently used pages will likely be used again.
  • Provides better performance than FIFO in many workloads.
  • Requires additional tracking of page usage history.

Example

Reference String: 1, 2, 3, 1, 4

Frames: [1,2,3] → page 1 used again → replace least recently used page (2).

Other Page Replacement Algorithms

MRU (Most Recently Used)

  • Replaces the most recently accessed page.
  • Useful in specific workloads such as sequential scans.

Random Replacement

  • Selects a page randomly for replacement.
  • Very simple but performance may vary.

Clock Algorithm

  • An efficient approximation of LRU.
  • Uses a circular list with reference bits.

Practical Example

Reference String: 1, 3, 0, 3, 5, 6

  • FIFO → 6 page faults
  • LRU → 5 page faults
  • Optimal → 4 page faults

This comparison shows how algorithm choice affects memory performance.

Performance Impact

  • Efficient algorithms reduce expensive disk I/O.
  • Poor algorithms may cause thrashing.
  • Modern OS use optimized LRU-based strategies.

Conclusion

Page replacement algorithms are essential for efficient memory management in modern operating systems.

  • They help maximize the use of limited RAM.
  • Different algorithms provide different performance trade-offs.
  • Understanding them helps design better computing systems.
